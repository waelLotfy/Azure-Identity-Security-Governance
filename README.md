## Labs Completed

---

## Core Concepts Explored

### Azure Key Vault
- Secure storage for secrets, keys, and certificates
- Centralized secret management
- Improves security and compliance

### Azure Role-Based Access Control (RBAC)
- Controls who can access Azure resources
- Uses predefined roles such as Virtual Machine Contributor
- Enforces least-privilege access

### Monitoring and Auditing
- Activity Logs track access and configuration changes
- Supports governance and compliance requirements

---

## Key Learnings
- Azure Key Vault protects sensitive information outside application code.
- Access to Azure resources is controlled using RBAC.
- Roles define what actions users can perform.
- Azure provides auditing and monitoring for access changes.
- Identity and governance are foundational to cloud security.
